Restaurant Management System 

Frontend Integration & API Contract v1
________________


System Architecture
Customer Mobile App
→ API Server
→ MongoDB
Admin Panel
→ API Server
→ MongoDB
All frontend applications communicate only through backend APIs.
Base URL:
/api/v1
________________


User Roles
Customer
Owner
Admin
Manager
Kitchen Staff
Delivery Staff
________________


Business Rules
* Single Restaurant Brand
* Maximum 9 Outlets
* Same Menu Across All Outlets
* Delivery Assignment is Manual
* No Rider App in Phase 1
* No Live GPS Tracking
* Estimated Delivery Time is manually entered by Admin
* Customer selects outlet before ordering
* Menu updates reflect across all outlets
* If outlet count exceeds 9:
Upgrade to Premium Plan
